Organising Files

Once you have downloaded a text editor, create a folder in Finder (if you're on Mac), preferably with the name of your website (e.g. first_website (underscores, lowercase, hyphens and no spaces are good in file naming)) Then, inside your main folder,create four folders as follows:

  • Scripts
  • Styles
  • Images
  • Videos

Notes:(As you'll see later, organising different file types into their respective folders is a good idea.)
do NOT create a "html" folder and move your html files in their, unless all the other folders are in the "html" folder as well. Make sure to keep your html files in the main folder (your website folder) so they are in the SAME folder as the scripts, styles, etc.

Next, open your text editor (application for editing code such as Visual Studio Code, Sublime Text, Bracketes or Notepad++ (if you're on Windows)) and create a new file with command+N or the File→new in the "File" menu at the top left corner of your screen (MacOS)

Once you've created your file, name it "index.html" which is the name for the main file when you publish it on Github. If it is not the 'homepage' or if you don't intend to publish it using Github, you can call it whatever you want as long as it has ".html" at the end to signify it is a HTML file. If so, avoid spaces or special characters in your file name.

If you want to use images and videos, move any images or videos into the "images" or "videos" folder. We can talk about the CSS files (styles) or JavaScript files (scripts) later.
